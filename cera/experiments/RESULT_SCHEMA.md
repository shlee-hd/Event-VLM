# CERA Result Schema

This document defines machine-readable outputs used to replace projected tables in the paper.

## 1) Per-run files

Path pattern:

- Main: `cera/experiments/results/main/<dataset>/<profile>/seed_<seed>/`
- Ablation: `cera/experiments/results/ablation/<dataset>/<ablation>/seed_<seed>/`

Required files:

- `metrics.json`: scalar metrics from one run
- `predictions.json`: per-video predictions for paired tests

## 2) Aggregated CSV files

Generated by `cera/experiments/scripts/collect_metrics.py`.

### `main_runs.csv` / `ablation_runs.csv`

- One row per `(dataset, setting, seed)` run.
- Base columns:
  - `dataset`
  - `setting`
  - `seed`
  - `run_dir`
- Dynamic metric columns:
  - numeric keys from `metrics.json` (e.g., `auc`, `ap`, `fps`, `token_reduction`)

### `main_summary.csv` / `ablation_summary.csv`

- Long format, one row per `(dataset, setting, metric)`.
- Columns:
  - `dataset`
  - `setting`
  - `metric`
  - `mean`
  - `std`
  - `n`

### `main_summary_wide.csv` / `ablation_summary_wide.csv`

- Wide format, one row per `(dataset, setting)`.
- Columns:
  - `dataset`
  - `setting`
  - `<metric>_mean`
  - `<metric>_std`

## 3) Ablation manifest

Generated by `cera/experiments/scripts/run_ablation.py`.

File:

- `cera/experiments/results/ablation/ablation_manifest.csv`

Columns:

- `dataset`
- `ablation`
- `seed`
- `status` (`completed`, `pending_runtime_support`, `failed`)
- `reason`
- `output_dir`

## 4) Significance report

Generated by `experiments/paired_significance.py`.

Path pattern:

- `cera/experiments/results/stats/significance/<dataset>/detr_vs_yolo/`

Files:

- `significance.json`
- `significance.md`

## 5) Paper table stubs

Generated by `cera/experiments/scripts/render_paper_tables.py`.

Files:

- `paper_table_stub.md`
- `paper_table_stub.tex`
